DATA SOURCES & HARMONIZATION SUMMARY

=====================================
1. DATA SOURCES
=====================================

Bank A — Sparkov (Fraud Detection)
  Kaggle slug:    kartik2112/fraud-detection
  Files:          fraudTrain.csv, fraudTest.csv
  Original size:  ~1,852,394 rows (train + test combined)
  Output size:    500,000 rows (subsampled to reduce size imbalance, preserving fraud rate)
  Fraud rate:     ~0.52%
  Key features:   Timestamp, amount, merchant category (14 types), demographics
                  (gender, DOB, job), geographic coordinates (card & merchant),
                  city population

Bank B — BankSim
  Kaggle slug:    ealaxi/banksim1
  Files:          bs140513_032310.csv
  Original size:  594,643 rows (used in full)
  Key features:   Time step, amount, merchant category (15 types), customer age
                  bracket, gender, customer zip code, merchant zip code

Bank C — CC Fraud Transactions
  Kaggle slug:    anurag629/credit-card-fraud-transaction-data
  Files:          CreditCardData.csv
  Original size:  100,000 rows (used in full)
  Key features:   Date, time, amount, merchant group, transaction country,
                  shipping/billing address, gender, age, card type,
                  entry mode (chip/tap/PIN), transaction type (online/POS/ATM)

=====================================
3. HARMONIZATION STEPS PER BANK
=====================================

--- Bank A (Sparkov) ---
  amount:           Used directly from raw "amt" column
  log_amount:       log(1 + amount)
  amount_zscore:    Z-score computed within Bank A's amount distribution
  amount_percentile: Percentile rank within Bank A
  hour/dow:         Extracted from transaction datetime, encoded cyclically (sin/cos)
  is_weekend:       Derived from day_of_week
  is_night:         Derived from hour_of_day
  is_round_amount:  Derived from amount
  gender_M:         Mapped from "gender" column
  age_normalized:   Computed from DOB, normalized to [0,1] (min 18, max 100)
  geo_encoded:      Euclidean distance between card lat/long and merchant lat/long,
                    clipped at 99th percentile, normalized to [0,1]
  cat_*:            Merchant "category" mapped via SPARKOV_CAT_MAP to 7 unified categories,
                    then one-hot encoded

--- Bank B (BankSim) ---
  amount:           Used directly
  log_amount:       log(1 + amount)
  amount_zscore:    Z-score computed within Bank B's amount distribution
  amount_percentile: Percentile rank within Bank B
  hour/dow:         BankSim uses a "step" column (integer time steps); mapped to
                    approximate hour/day, then encoded cyclically
  is_weekend:       Derived from day_of_week
  is_night:         Derived from hour_of_day
  is_round_amount:  Derived from amount
  gender_M:         Mapped from gender column
  age_normalized:   Age bracket strings (e.g. "2", "3") mapped to midpoint ages
                    via AGE_BRACKET_MAP: {0:20, 1:30, 2:40, 3:50, 4:60, 5:70, 6:80},
                    then normalized to [0,1]
  geo_encoded:      Absolute difference between customer zip code and merchant zip code,
                    clipped at max, normalized to [0,1]
  cat_*:            "category" column (es_food, es_fashion, etc.) mapped via
                    BANKSIM_CAT_MAP to 7 unified categories, then one-hot encoded

--- Bank C (CC Fraud Transactions) ---
  amount:           Parsed from string with currency symbol (e.g. "£5"), converted to float
  log_amount:       log(1 + amount)
  amount_zscore:    Z-score computed within Bank C's amount distribution
  amount_percentile: Percentile rank within Bank C
  hour/dow:         Extracted from "Date" + "Time" columns combined into datetime,
                    encoded cyclically
  is_weekend:       Derived from day_of_week
  is_night:         Derived from hour_of_day
  is_round_amount:  Derived from amount
  gender_M:         Mapped from "Gender" column
  age_normalized:   "Age" column normalized to [0,1] (min 18, max 100);
                    defaults to 0.5 if column missing
  geo_encoded:      "Country of Transaction" label-encoded (each unique country gets
                    a numeric code), then normalized to [0,1]; defaults to 0.5 if missing
  cat_*:            "Merchant Group" column mapped via CCFRAUD_CAT_MAP to 7 unified
                    categories, then one-hot encoded


=====================================
4. MERCHANT CATEGORY MAPPING
=====================================

7 unified categories: grocery, shopping, entertainment, gas_transport,
                      food_dining, health_personal, other

Sparkov -> Unified:
  grocery_pos, grocery_net           -> grocery
  shopping_pos, shopping_net         -> shopping
  misc_pos, misc_net                 -> shopping
  entertainment, travel              -> entertainment
  gas_transport                      -> gas_transport
  food_dining                        -> food_dining
  health_fitness, personal_care      -> health_personal
  home, kids_pets                    -> other

BankSim -> Unified:
  es_food, es_hyper                  -> grocery
  es_sportsandtoys, es_fashion, es_tech -> shopping
  es_hotelservices, es_leisure, es_travel -> entertainment
  es_transportation                  -> gas_transport
  es_barsandrestaurants              -> food_dining
  es_health, es_wellnessandbeauty    -> health_personal
  es_otherservices, es_contents, es_home -> other

CC Fraud -> Unified:
  grocery, food, supermarket         -> grocery
  shopping, retail, electronics, clothing, fashion -> shopping
  entertainment, travel, hotel, leisure -> entertainment
  gas, transport, transportation     -> gas_transport
  restaurant, dining, bar            -> food_dining
  health, beauty, personal           -> health_personal
  (anything else)                    -> other
